# Agent Instructions for `proto`

## Purpose

The `proto` crate contains **auto-generated Rust code from protocol buffer definitions**. It provides type-safe Rust types and gRPC stubs for communicating with Pulumi's runtime.

**What it does:** Generates Rust types and gRPC clients/servers from `.proto` files.

## Architecture Concepts

### Runtime Protocol Module
The crate generates code for the **`pulumi` module** used for talking to the Pulumi runtime
(resource monitor and engine) via gRPC.

### Code Generation Process
The code is NOT hand-written. It's generated by the `regenerator` crate using `tonic-build` and `prost`:
- Read `.proto` files
- Generate Rust structs for messages
- Generate gRPC client/server code (optional)
- Output to `src/` directory

### Protocol Buffer Benefits
Using protobuf provides:
- Cross-language compatibility with Pulumi's Go runtime
- Efficient binary serialization
- Backward compatibility through versioned schemas
- Type safety via generated code

## Key Concepts

- **Generated code**: Never edit `.rs` files directly - edit `.proto` files instead
- **Feature flags**: `pulumi` enables gRPC client/server bindings
- **Regeneration**: Run `cargo run -p regenerator` after changing `.proto` files
- **Two protocols**: Runtime communication vs schema serialization

## When to Modify

Modify this crate when:
- Pulumi's protocol buffer definitions change (update from upstream)
- Pulumi adds new services or message types
- Need to regenerate code due to `prost`/`tonic` version updates

**Important**: Modify `.proto` files, not generated `.rs` files!

## Testing Philosophy

This crate is mostly generated code, so testing happens in consuming crates:
- `grpc` crate tests the gRPC client/server functionality
- `schema_protobuf` crate tests schema serialization

The generation process itself is validated by successful compilation.

## Integration Points

- **Used by `grpc`**: Imports gRPC clients and protocol types (with `pulumi` feature)
- **Generated by `regenerator`**: Regeneration tool that runs `tonic-build`
